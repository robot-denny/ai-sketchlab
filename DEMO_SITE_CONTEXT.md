### MCP Configuration Notes

**Two Config Systems:**
- **Standalone Claude Code:** Uses `~/.claude.json` with project-specific MCP configs
- **Claude Code in VS Code:** Uses `.mcp.json` in project root

**Current Setup:**
- `.mcp.json` references environment variables from `.env` file
- Credentials: Uses "mcp-dev" API user (becomes active after first successful connection)
- Collections enabled: `document`, `media`, `document-type`, `data-type`

**Known Behaviors:**
- `/doctor` warnings about missing env vars can be ignored if MCP works at runtime
- The diagnostic tool does static checking and doesn't detect `.env` file loading
- MCP tools context is large (~45k tokens from 128 tools) but manageable
- New API users show "Inactive" until first successful MCP connection

**Troubleshooting:**
- Ensure Umbraco site is running: `cd src/UmbracoProject && dotnet run`
- Verify credentials in `.env` file match active API user in Umbraco backoffice
- Restart Claude Code after `.mcp.json` changes
- Check MCP status: ask Claude Code to run `/doctor`
