# Roadmap

The project-level queue. Each entry is a slug, a one-line intent, and a pointer to the feature folder, PRD, or "no spec yet."

Per-feature increments (shipped + planned within a feature) live in the **Increments** section of each `_features/<slug>.md`. This file tracks work *across* features and the things that don't yet belong to a feature.

When a body of work spans three or more features and needs a shared intent doc, write a PRD at `_prds/<slug>.md` and link it from the **Bundles** section below.

**Pillar 2 architecture push.** All items below feed a single goal: move Pillar 2 (architectural separation) from 2 ‚Üí 4 by adopting the **agency-standard solution architecture** (reference: [dev-kittitas-county](_audits/2026-05-19-dev-kittitas-county.md)). **Destination, decided after studying the reference (2026-06-19):** a two-project **RCL split** ‚Äî a new `UmbracoProject.Features` Razor Class Library holding business logic, with `UmbracoProject` as the thin host ‚Äî organized **folder-by-kind** (`Abstractions/ Services/ Composer/ Constants/ Extensions/ Infrastructure/ Models/ Controllers/`, vertical slices inside `Blocks/ Pages/ Controllers/API/<Domain>/`). This **supersedes the earlier `src/UmbracoProject/Features/<FeatureName>/` premise** ‚Äî the reference uses no such folder. Goal is architectural *resilience* + recognizability to the team, **not** codebase parity: the reference's embedded-RCL-views + `ViewModelFactory`/`TemplateCoordinator` rendering framework is **declined** (parity-only, low marginal resilience); views stay in the host's stock `Views/`. The safety net (CI + screenshots + warnings-as-errors) shipped as `arch-safety-net`. See the [2026-05-19 audit](_audits/2026-05-19-umbraco-17-demo-site.md) (Pillar 2 = 2/5) and the [feature-folder-architecture spec](_specs/shipped/feature-folder-architecture.md) for the full rationale and the three-tier adopt/defer/decline decision.

Independently unblocks build-time Razor obsolete-API detection, retiring `[[project_inmemoryauto_blocks_buildtime_razor.md]]` and de-risking `arch-obsolete-api-migration`. **Sequenced immediately after the pilot, before any model-coupled migration slice** ‚Üí no spec yet
