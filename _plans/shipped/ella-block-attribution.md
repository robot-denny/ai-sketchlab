# Plan: Ella Block Attribution

**Spec**: `_specs/ella-block-attribution.md`
**Branch**: `claude/feature/ella-block-attribution`

## Context

The blog is becoming a co-writing surface between a human site author and an AI persona named "Ella". The design system already ships the visual treatment (`.ella` rules in [article-page.css:49-53](src/UmbracoProject/wwwroot/assets/css/article-page.css#L49-L53)), and the existing per-block `Author` picker on `richTextRow` / `imageRow` / `latestArticlesRow` already drives a quiet `<p class="block-author">— Name</p>` byline. What's missing is the connection between the two: nothing today marks an Author as an AI persona, and nothing in the orchestrator partial knows to swap the quiet byline for the strong inline-note treatment.

This plan adds **one boolean property** (`isAi`) to the Author doc type via a small reusable composition, **one wrapper** in the blocklist orchestrator, and **one CSS rule set** that styles the wrapper. No per-block-component partial is touched. The wrapper is `display: contents` (layout-neutral), the eyebrow text is generated by CSS `::before` using a CSS custom property the orchestrator writes inline, and the trailing `.block-author` byline is suppressed by a CSS descendant selector so the same name never appears twice in one card.

---

## Key Decisions

- **`isAi` lives on a new composition `AIPersonaProperties`** (not directly on the Author doc type). Follows the established `HideProperty` / `SpacingProperties` precedent in [document-type__02180d871eea45d993a61e38f2ee0165.uda](src/UmbracoProject/umbraco/Deploy/Revision/document-type__02180d871eea45d993a61e38f2ee0165.uda); `isAi` becomes reusable if a future doc type (e.g. member, contributor) also needs a persona flag, and a "Persona" tab on the Author doc type discourages accidental flips. Confirmed in open question #6.
- **Reuse the built-in `True/false` data type** (id `92897bc6-a5f3-4ffe-ae27-f2e7e33dda49`, editor `Umb.PropertyEditorUi.Toggle`) — same data type `HideProperty.hide` uses. No new data type needed.
- **Wrapper element, not a per-component class** (confirmed open question #2). Orchestrator emits `<div class="ella-wrap" data-attributed-to="…" style="--persona-name: '…'">…partial output…</div>` only when treatment is active. Otherwise no wrapper, no attribute, no DOM diff vs. today.
- **`display: contents` on the wrapper** so layout is unchanged. `.ella` styling (warm surface, left border, light body weight) is applied to `.ella-wrap[data-attributed-to] > *` — i.e. the inner block's `<div class="richtext">` / `<div class="image">` element — so existing block CSS continues to apply.
- **Eyebrow via CSS `::before` + CSS custom property** for persona name. CSS:
  ```css
  .ella-wrap[data-attributed-to] > *::before {
    content: "Written by " var(--persona-name) " · inline note";
  }
  @media (max-width: 760px) {
    .ella-wrap[data-attributed-to] > *::before {
      content: var(--persona-name) " · inline note";
    }
  }
  ```
  Razor emits `style="--persona-name: 'Ella'"` so a future second persona renders correctly without code changes (confirmed open question #7). CSS variables cascade through `display: contents`, so the inner block's `::before` reads the wrapper's variable. Narrow-viewport collapse to short form per open question #1.
- **Suppress trailing `.block-author` via CSS**, not by changing per-block partials: `.ella-wrap[data-attributed-to] .block-author { display: none; }`. Honors acceptance criterion 6 ("touches one orchestrator partial, one CSS file, and the Author doc type — not the eleven individual block component partials").
- **Article-author detection in the orchestrator**, not in each partial. `Umbraco.AssignedContentItem?.Value<IEnumerable<IPublishedContent>>("author")` is the article authors collection (null/empty on non-article pages — wrapper simply never activates there). Suppress per-block wrapper only when *all* article authors are `isAi` (confirmed open question #3).
- **`data-attributed-to` value = author's `UrlSegment`** (slug form, e.g. `"ella"`). Stable identifier suitable for CSS hooks and per-persona overrides if ever needed.
- **No C# helper class**, no new ViewImports import. The wrapper decision is ~6 lines in `default.cshtml`; extracting a helper is premature given there's exactly one call site. Skips the xUnit suggestion in the spec — the four-row truth table is covered fully by the Playwright E2E.
- **Schema is created via a throwaway Management API script** under `scripts/setup-ella-attribution-schema.mjs`, following the precedent of [scripts/setup-guides-schema.mjs](scripts/setup-guides-schema.mjs). Script is deleted after running and `.uda` files are committed.

**What to build**:

**[Views/Partials/blocklist/default.cshtml](src/UmbracoProject/Views/Partials/blocklist/default.cshtml)** — replace the foreach with:
```razor
@inherits Umbraco.Cms.Web.Common.Views.UmbracoViewPage<Umbraco.Cms.Core.Models.Blocks.BlockListModel>
@using Umbraco.Cms.Core.Models.PublishedContent
@{
    if (Model?.Any() != true) { return; }

    // Resolve article-level authors once (null on non-article hosts → wrapper never activates)
    var articleAuthors = Umbraco.AssignedContentItem?.Value<IEnumerable<IPublishedContent>>("author");
    var articleIsAllAi = articleAuthors != null
        && articleAuthors.Any()
        && articleAuthors.All(a => a.Value<bool>("isAi"));
}
<div class="umb-block-list">
    @foreach (var block in Model)
    {
        if (block?.ContentKey == null) { continue; }
        var data = block.Content;
        var blockAuthor = data.Value<IPublishedContent>("author");
        var wrap = blockAuthor != null
            && blockAuthor.Value<bool>("isAi")
            && !articleIsAllAi;

        if (wrap)
        {
            // CSS escape: single-quote → \27 ; backslash → \\
            var personaName = blockAuthor.Name?.Replace("\\", "\\\\").Replace("'", "\\27 ") ?? "";
            var personaSlug = blockAuthor.UrlSegment ?? blockAuthor.Name?.ToLowerInvariant().Replace(" ", "-") ?? "ai";
            <div class="ella-wrap" data-attributed-to="@personaSlug" style="--persona-name: '@Html.Raw(personaName)';">
                @await Html.PartialAsync("blocklist/Components/" + data.ContentType.Alias, block)
            </div>
        }
        else
        {
            @await Html.PartialAsync("blocklist/Components/" + data.ContentType.Alias, block)
        }
    }
</div>
```

**[wwwroot/assets/css/article-page.css](src/UmbracoProject/wwwroot/assets/css/article-page.css)** — append under the `.ella` block:
```css
/* AI persona attribution wrapper — applied via <div class="ella-wrap" data-attributed-to>
   around any block whose Author is an isAi persona inside a non-all-AI article. */
.ella-wrap[data-attributed-to]{display:contents}
.ella-wrap[data-attributed-to] > *{margin:2em 0;padding:20px 24px;background:var(--surface-secondary);border-left:2px solid var(--text-tertiary);font-weight:300}
.ella-wrap[data-attributed-to] > *::before{content:"Written by " var(--persona-name) " · inline note";display:flex;align-items:center;gap:8px;font-family:var(--font-mono);font-size:10.5px;letter-spacing:.1em;text-transform:uppercase;color:var(--text-secondary);margin-bottom:8px}
.ella-wrap[data-attributed-to] > *::before{}
.ella-wrap[data-attributed-to] > *::before::before{content:"";width:6px;height:6px;background:var(--text-tertiary);border-radius:50%}
.ella-wrap[data-attributed-to] .block-author{display:none}
@media (max-width:760px){
  .ella-wrap[data-attributed-to] > *::before{content:var(--persona-name) " · inline note"}
}
```
**Note on the bullet-dot**: CSS does not allow `::before::before`. Drop the nested-pseudo line and instead inline the dot into the eyebrow string: `content: "● " var(--persona-name) " · inline note"`. Or use a left padding + a `background-image` with a tiny SVG circle. Choose at implementation time — small visual detail, not a behavioral requirement.
